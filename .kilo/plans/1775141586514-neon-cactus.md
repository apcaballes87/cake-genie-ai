# Plan: Fix 643 SEranking Audit Issues for genie.ph

## Executive Summary
Based on a codebase audit, I've identified key areas causing SEO issues. The 643 issues likely span multiple categories (technical, on-page, content, images). Below is a prioritized plan to diagnose and fix these issues.

---

## Step 1: Create robots.txt (Critical - Likely ~50+ issues)
**Problem**: No `robots.txt` file exists in the project
**Impact**: Missing crawler instructions, sitemap not declared, potential crawl budget issues

**Solution**:
- Create `/public/robots.txt` with:
  - Sitemap declaration
  - Allow crawling of public pages
  - Block admin/API routes
  - Add AI crawler rules (Google-Extended, GPTBot, OAI-SearchBot, ClaudeBot)

---

## Step 2: Core Web Vitals Fixes (Likely ~150+ issues)
**Problem**: Images are unoptimized (`unoptimized: true` in next.config.ts)
**Impact**: Poor LCP (Largest Contentful Paint), high CLS (Cumulative Layout Shift)

**Solution**:
- Enable Next.js built-in image optimization (remove `unoptimized: true`)
- Or use Cloudinary/Imgix for external optimization
- Add explicit width/height to all images (prevents CLS)
- Implement lazy loading with `priority` for above-fold images

---

## Step 3: Image Alt Text Audit (Likely ~200+ issues)
**Problem**: Some images have generic or missing alt text (found 166+ patterns using `alt={}`)
**Impact**: Accessibility failures, lost image search traffic

**Solution**:
- Audit all images using grep pattern: `alt={}` (empty alt)
- Add descriptive alt text to decorative images (e.g., "Birthday cake with pink frosting")
- Ensure product images have keyword-rich alt text

---

## Step 4: Duplicate Content & Canonical Issues (Likely ~100+ issues)
**Problem**: Dynamic pages at scale (`/customizing/[slug]`, `/designs/[slug]`)
**Impact**: Potential duplicate content, diluted page authority

**Solution**:
- Verify canonical URLs are properly set on all dynamic pages
- Check for proper handling of legacy slug redirects (already implemented in `/customizing/[slug]/page.tsx`)
- Ensure each design has unique title/description meta tags

---

## Step 5: Schema Markup Validation (Likely ~50+ issues)
**Problem**: JSON-LD schemas may have missing fields or syntax errors
**Impact**: Failed rich results, missing schema in search

**Solution**:
- Add missing fields to LocalBusiness schema (priceRange, servesCuisine)
- Add Product schema on product pages
- Add FAQPage schema (commented as deprecated but still useful)
- Validate all schemas with Google Rich Results Test

---

## Step 6: Mobile Usability (Likely ~50+ issues)
**Problem**: Tap targets too close, viewport issues
**Impact**: Mobile-friendly penalty in Google

**Solution**:
- Ensure buttons/links have 48px minimum tap target
- Add proper viewport meta tag (already present in layout.tsx)
- Test with Google Mobile-Friendly Test

---

## Step 7: Meta Tags Audit
**Action**: Review pages missing unique meta descriptions

**Solution**:
- Verify all pages have unique title and description
- Check dynamic pages pull from database fields (already implemented)
