# Plan: Fix "Cannot read image" Error in AI Image Editing

## Executive Summary
The error "Cannot read image (this model does not support image input)" occurs when users try to use the AI image editing feature. This is a Gemini API error. The system uses `gemini-2.5-flash-image` for image editing.

---

## Root Cause Analysis

The error likely originates from one of these issues:

### 1. **Model Quota Limits (Most Likely)**
- Google AI Studio has rate limits on image generation models
- The error message "Cannot read image" sometimes appears when quota is exceeded
- Check: Google AI Studio quota page

### 2. **API Key Configuration**
- The environment variable `GOOGLE_AI_API_KEY` must be valid
- Server-side vs client-side key usage
- Check: `/src/lib/ai/client.ts` line 8

### 3. **Image Data Format Issues**
- Base64 encoding might be malformed
- MIME type mismatches
- Check: `/src/app/api/ai/edit-image/route.ts` lines 77-83

### 4. **Model Version Mismatch**
- `gemini-2.5-flash-image` might be deprecated or renamed
- Alternative: `gemini-1.5-pro` with vision capabilities

---

## Investigation Steps

### Step 1: Check Google AI Studio Quota
- Go to https://aistudio.google.com/app/quota
