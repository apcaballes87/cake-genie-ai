# Custom Cake Collection SEO Blueprints

Research date: 2026-06-02

## Research Notes

This launch set focuses on missing collection-page gaps that can support transactional Cebu cake orders and match the existing design cache. Existing collections such as `vintage-cake`, `minimalist-cake`, `drip-cake`, `debut-cake`, `anniversary-cake`, `gender-reveal-cake`, `money-cake`, `floral-cake`, `ribbon-cake`, and `lambeth-cake` should be improved in place rather than duplicated.

Philippines keyword data supports the milestone and aesthetic opportunities. DataForSEO's Google keyword dataset reported `1st birthday cake` at 390 average monthly searches with transactional intent and `vintage cake design` at 1,000 average monthly searches. Local SERP evidence also shows Cebu-facing listings for pink minimalist cakes, Korean-inspired ombre minimalist cakes delivered in Cebu, and custom birthday, debut, wedding, and anniversary cakes. Trend sources continue to identify minimalist, muted-color, vintage, Lambeth, and bento styling as active cake-design patterns.

Exact Cebu-only search volume is not available from the connected dataset. The collection focus keywords below therefore combine Philippines-level keyword signals with explicit Cebu transactional modifiers.

## FAQ Rendering Note

Collection pages should render helpful visible FAQ accordions for shoppers. Keep the existing `CollectionPage`, `ItemList`, and `BreadcrumbList` structured data. Do not add `FAQPage` JSON-LD for commercial collection pages.

## Image Synchronization Rules

The synchronizer should match each row in `cakegenie_analysis_cache` using the collection's core name plus the multi-word tag variants in the SQL migration. The existing script checks:

1. Exact lowercase cache `tags` array containment for the cleaned collection name.
2. Case-insensitive `keywords` text matches for the cleaned collection name.
3. Case-insensitive `slug` matches for the cleaned collection name.
4. Exact lowercase cache `tags` array containment for each multi-word collection tag.

For better recall, ensure analysis cache tags use the normalized variants from the migration. Color collections should carry plain color and cake phrase tags. Style collections should include the visual finish terms. Recipient and milestone pages should include both numeric and written variants where applicable.

The current script counts rows with `original_image_url`, `slug`, and `price`, then prefers a non-empty `studio_edited_image_url` for `sample_image`. It falls back to `original_image_url` only when no verified studio-edited match is available.

1. Apply `supabase/migrations/20260602090000_add_custom_cake_merchandising_collections.sql`, then `supabase/migrations/20260602103000_add_collection_quality_metadata.sql`, to the target Supabase project.
2. Run a preview sync:

   ```bash
   npx tsx scripts/update-collections-studio-images.ts --dry-run
   ```

3. Review match counts and thumbnails. Any zero-match collection should remain unpublished until cache tags or metadata are improved.
4. Populate collection images and counts:

   ```bash
   npx tsx scripts/update-collections-studio-images.ts
   ```

5. Verify that each launched slug returns designs and a `200` response before adding internal links.
6. Keep visible FAQ blocks for shoppers, without adding commercial `FAQPage` JSON-LD.
7. Add contextual internal links from `/collections`, relevant blog articles, and nearby existing collection pages.
